## 核心知识点

### Prompt 四要素模型

```
┌────────────────────────────────────────────┐
│  1. 角色设定 (Role)                         │
│     "你是一位资深的 Python 开发工程师"       │
├────────────────────────────────────────────┤
│  2. 任务指令 (Task)                         │
│     "请帮我优化以下代码的性能"               │
├────────────────────────────────────────────┤
│  3. 上下文信息 (Context)                    │
│     "这是一个处理百万级数据的批处理脚本"     │
├────────────────────────────────────────────┤
│  4. 输出格式 (Format)                       │
│     "请按照：分析 → 方案 → 代码格式输出"    │
└────────────────────────────────────────────┘
```

### 设计模式一览

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| 指令模式 | 直接明确的任务指令 | 简单任务 |
| 角色模式 | 设定专业角色 | 专业领域 |
| 模板模式 | 提供结构化模板 | 格式化输出 |
| 约束模式 | 添加限制条件 | 精确控制 |
| 分解模式 | 复杂任务分步 | 多步骤任务 |
| 对比模式 | 比较分析 | 决策问题 |
| Few-shot | 提供示例引导 | 特定格式 |
| CoT | 展示推理过程 | 逻辑推理 |

### Chain-of-Thought 技术

```
Zero-shot CoT:
问题 + "让我们一步一步思考" → 更好的推理结果

Few-shot CoT:
问题 + 推理示例 + 新问题 → 高质量推理输出

Self-Consistency:
多次采样 → 投票选择最优答案
```
