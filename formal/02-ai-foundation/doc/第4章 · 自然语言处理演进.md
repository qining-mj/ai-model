## 4.1 NLP 任务概览

### 什么是自然语言处理

> **NLP（Natural Language Processing）= 让计算机理解和生成人类语言**

```
人类语言 ←───── NLP ─────→ 计算机理解

"今天天气真好"  →  情感: 正面
                 →  实体: 今天(时间), 天气(主题)
                 →  意图: 闲聊/天气查询
```

### 核心任务

| 任务 | 说明 | 示例 |
|------|------|------|
| **文本分类** | 判断文本类别 | 垃圾邮件识别、情感分析 |
| **命名实体识别 (NER)** | 识别人名、地名等 | "马云在杭州" → 马云(人名), 杭州(地名) |
| **机器翻译** | 语言转换 | 英文 → 中文 |
| **问答系统** | 回答问题 | "中国首都是？" → "北京" |
| **文本生成** | 生成文本 | 写作、对话 |

### 为什么 NLP 很难

```
歧义问题
─────────────────────────────────
"我用望远镜看到了他"
  → 我用望远镜看，看到了他
  → 我看到他拿着望远镜

上下文依赖
─────────────────────────────────
"苹果很好吃" → 水果
"苹果发布新手机" → 公司

常识推理
─────────────────────────────────
"他把冰箱打开，把大象放进去" → 需要理解这是个笑话
```

## 4.2 传统 NLP 方法

### 词袋模型（Bag of Words）

**把文本看作词的集合，忽略顺序**

```
文档1: "我 爱 北京"     →  {我:1, 爱:1, 北京:1, 天安门:0}
文档2: "我 爱 天安门"   →  {我:1, 爱:1, 北京:0, 天安门:1}

问题：忽略了词序
"狗咬人" 和 "人咬狗" 表示相同 ← 这明显不对
```

### TF-IDF

**词频（TF）× 逆文档频率（IDF）**

```
TF = 词在文档中出现次数 / 文档总词数
IDF = log(文档总数 / 包含该词的文档数)

"的" → 出现频繁但不重要 → TF高 × IDF低 = 低分
"Transformer" → 出现少但有区分度 → TF适中 × IDF高 = 高分
```

### N-gram 语言模型

**用前 N-1 个词预测下一个词**

```
2-gram (Bigram):
P(北京 | 我爱) ← 给定"我爱"，下一个词是"北京"的概率

3-gram (Trigram):
P(广场 | 天安门) ← 给定"天安门"，下一个词是"广场"的概率
```

### 传统方法的局限

| 问题 | 说明 |
|------|------|
| **稀疏性** | 词表巨大，大多数组合未见过 |
| **无语义** | "狗"和"犬"被当作完全不同的词 |
| **无上下文** | 同一个词在不同语境含义不同 |

## 4.3 词向量革命

### Word2Vec：词的分布式表示

> **核心思想：一个词的含义由它的上下文决定**

```
训练方式：
"猫 坐在 [沙发] 上"  →  预测中间词
"狗 坐在 [沙发] 上"  →  猫和狗的向量会很接近
"我 坐在 [沙发] 上"

结果：语义相近的词，向量距离也近
```

### 词向量的魔力

```
向量运算可以表达语义关系！

king - man + woman ≈ queen
                                    
    king ●─────────→ queen
         │          │
     man ↓          ↓ woman
         ●─────────→●

北京 - 中国 + 日本 ≈ 东京
```

### 静态词向量的局限

**一词多义问题**

```
"苹果" 只有一个向量，但含义不同：

"苹果很甜"      → 水果
"苹果发布会"    → 公司

静态词向量无法区分！
→ 需要动态的、上下文相关的词表示
```

## 4.4 从 RNN 到 Transformer

### RNN/LSTM：处理序列

```
RNN 的递归结构

    h₁ → h₂ → h₃ → h₄ → h₅
    ↑    ↑    ↑    ↑    ↑
   "我" "爱" "自" "然" "语言"

每一步都考虑之前的隐藏状态（记忆）
```

**问题**：
- 长距离依赖难以学习（梯度消失）
- 无法并行计算（必须顺序处理）

### Seq2Seq + Attention

**机器翻译的突破**

```
编码器                      解码器
"I love you"              "我 爱 你"
    ↓                         ↓
  [编码]     Attention     [解码]
    ↓          ↓↓↓           ↑
  向量表示  ←─关注哪里─→   逐词生成

Attention 让解码器知道该关注源句子的哪个部分
```

### Transformer 诞生

**2017年 Google 论文：Attention Is All You Need**

```
核心创新：完全基于 Attention，抛弃 RNN

传统 RNN                    Transformer
───────────────────────────────────────────
顺序处理                    并行处理
长距离依赖困难               直接建模任意距离
速度慢                      速度快

    Self-Attention: 每个词都能直接关注其他所有词
```

### 为什么 Transformer 统治一切

| 优势 | 说明 |
|------|------|
| **并行化** | 所有位置同时计算，训练快 |
| **长距离依赖** | 任意两个位置直接交互 |
| **可扩展** | 参数量越大效果越好 |
| **通用性** | NLP、CV、多模态都适用 |

## 4.5 预训练语言模型时代

### BERT：双向理解

```
预训练任务：遮住词，预测被遮住的词

输入: "我 爱 [MASK] 京"
目标: 预测 [MASK] = "北"

双向：同时利用左边和右边的上下文
     我 爱 ← [北] → 京
```

**特点**：擅长理解任务（分类、NER）

### GPT：生成式预训练

```
预训练任务：根据前文预测下一个词

输入: "我 爱 北"
目标: 预测下一个词 = "京"

单向（自回归）：只看前面，从左到右生成
     我 → 爱 → 北 → [京]
```

**特点**：擅长生成任务（写作、对话）

### BERT vs GPT

```
            BERT                      GPT
          (编码器)                   (解码器)
            
输入 → [双向 Attention] → 表示   输入 → [单向 Attention] → 生成
            
       "理解型选手"                  "生成型选手"
       
分类、问答、NER                  文本生成、对话、创作
```

### 从 BERT 到 ChatGPT

```
2018: BERT   → 预训练革命，理解任务
2018: GPT-1  → 生成式预训练
2019: GPT-2  → 规模扩大，零样本能力
2020: GPT-3  → 175B参数，涌现能力
2022: ChatGPT → RLHF对齐，走向大众
2023: GPT-4  → 多模态，推理增强
```
