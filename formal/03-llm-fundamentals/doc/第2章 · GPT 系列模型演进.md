## 2.1 GPT-1：预训练 + 微调范式的开创

### 核心创新

```
GPT-1 (2018年，OpenAI)

两阶段训练范式：
┌─────────────────────────────────────────────┐
│ 阶段1：无监督预训练                          │
│   在海量文本上学习语言规律                   │
│   任务：预测下一个词                         │
│   数据：BookCorpus（约5GB文本）              │
└─────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────┐
│ 阶段2：有监督微调                            │
│   在特定任务上适配                           │
│   任务：分类、问答等                         │
│   数据：少量标注数据即可                     │
└─────────────────────────────────────────────┘
```

### Decoder-Only 架构

```
为什么选择 Decoder-Only？

Encoder (BERT风格)           Decoder (GPT风格)
──────────────────────────────────────────────
双向注意力                   单向注意力（只看前面）
擅长理解                     擅长生成
完形填空                     从左到右续写

GPT 的目标是生成，所以选择 Decoder-Only
```

### 自回归生成

```
输入: "今天天气"
        ↓
GPT-1: 预测下一个词 → "很"
        ↓
输入: "今天天气很"
        ↓
GPT-1: 预测下一个词 → "好"
        ↓
...循环直到结束

每次只预测一个词，把预测结果加入输入，继续预测
```

### 参数规模

| 指标 | GPT-1 |
|------|-------|
| 参数量 | 1.17亿 (117M) |
| 层数 | 12 |
| 隐藏维度 | 768 |
| 注意力头 | 12 |

## 2.2 GPT-2：规模的力量

### 关键升级

```
GPT-2 (2019年)

参数量: 117M → 1.5B (扩大 13 倍)

发现：规模扩大带来质变
  - 不微调也能做下游任务（Zero-Shot）
  - 生成的文本更加连贯
  - 出现了"理解"的迹象
```

### Zero-Shot 能力涌现

```
传统方式                      GPT-2
──────────────────────────────────────────────
预训练 → 微调 → 使用          预训练 → 直接使用

需要标注数据                   不需要标注数据
每个任务单独微调               一个模型做多个任务

示例：
输入: "Translate English to French: Hello →"
GPT-2: "Bonjour"

没有见过翻译训练数据，却能翻译！
```

### WebText 数据集

```
数据来源：Reddit 高赞链接的网页内容
数据量：约 40GB
特点：
  - 质量高（经过社区筛选）
  - 多样性好（各种话题）
  - 排除了恶意内容
```

### "太危险而不敢发布"

```
OpenAI 的顾虑：
  - 生成的假新闻可能被滥用
  - 钓鱼邮件、虚假评论
  - 冒充他人写作

最终分阶段发布：
  小模型 → 中模型 → 大模型 → 完整版
```

## 2.3 GPT-3：In-Context Learning 的发现

### 参数量跃升

```
GPT-3 (2020年)

参数量: 175B (1750亿)

对比:
GPT-1:   117M  ▪
GPT-2:   1.5B  ▪▪▪▪▪▪▪▪▪▪▪▪▪
GPT-3:   175B  ▪▪▪▪▪▪▪▪▪▪▪▪▪ × 100+
```

### Few-Shot Learning：无需微调

```
Zero-Shot (零样本):
  输入: "情感分析：这部电影太棒了。情感："
  输出: "正面"

One-Shot (单样本):
  输入: "这本书很无聊。情感：负面
        这部电影太棒了。情感："
  输出: "正面"

Few-Shot (少样本):
  输入: "这本书很无聊。情感：负面
        景色真美。情感：正面
        服务太差了。情感：负面
        这部电影太棒了。情感："
  输出: "正面"

给几个例子，模型就能学会！
```

### 涌现能力

```
规模带来质变

小模型：只会简单模式匹配
中模型：能理解一些规则
大模型：涌现出推理、类比、创作能力

              能力
               │
               │                    ·涌现
               │                  ·
               │               ·
               │           ·
               │      ·····
               │·····
               └────────────────────→ 参数量
                                   突然提升
```

### Prompt 成为新的编程方式

```
传统编程: 写代码告诉计算机怎么做
Prompt:  用自然语言告诉模型要做什么

示例：

GPT-3 输出: 完整的代码

改变了人机交互的方式
```

## 2.4 InstructGPT 与 ChatGPT：RLHF 的突破

### 问题：GPT-3 并不"听话"

```
GPT-3 的问题：
  - 不遵循指令（答非所问）
  - 生成有害内容
  - 编造虚假信息
  - 不符合人类期望

原因：预训练目标是"预测下一个词"
     不是"给出有帮助的回答"
```

### 指令微调（Instruction Tuning）

```
第一步：收集指令-回答数据

人类标注员编写：
  指令: "解释什么是机器学习"
  回答: "机器学习是一种让计算机从数据中学习的技术..."

用这些数据微调模型，让它学会"遵循指令"
```

### RLHF：人类反馈强化学习

```
RLHF 三阶段

阶段1: 监督微调 (SFT)
  用指令-回答数据微调

阶段2: 训练奖励模型 (RM)
  人类对多个回答排序
  模型学习人类偏好
  
  回答A > 回答B > 回答C
       ↓
  奖励模型学会打分

阶段3: 强化学习优化 (PPO)
  用奖励模型的分数作为奖励
  优化语言模型生成高分回答
```

```
                    RLHF 流程图
                    
用户提问 ─────→ 语言模型 ─────→ 生成回答
                  ↑                │
                  │                ↓
                  │           奖励模型
                  │                │
                  └────── 强化学习 ←┘
                        (提高奖励分数)
```

### 为什么 RLHF 有效

| 方面 | 改进 |
|------|------|
| **有帮助** | 直接回答问题，不绕圈子 |
| **真实性** | 减少编造，承认不知道 |
| **无害性** | 拒绝有害请求 |

## 2.6 规模定律（Scaling Laws）

### Kaplan 定律

```
OpenAI 2020年发现：

性能 ∝ 参数量^0.076 × 数据量^0.095 × 计算量^0.050

含义：
- 参数量、数据量、计算量都重要
- 三者同时扩大效果最好
- 存在幂律关系
```

### Chinchilla 定律

```
DeepMind 2022年发现：

最优训练策略：
  参数量 和 数据量 应该同比例扩大

Chinchilla (70B) 用更多数据训练
效果超过 Gopher (280B)

结论：之前的模型普遍"训练不足"
```

### Scaling Laws 的意义

```
                    可预测的进步
                    
投入更多资源 ─────→ 性能可预测地提升

这让大公司敢于投资训练更大的模型
因为效果是"可预期"的
```
