## 2.1 什么是机器学习

### 传统编程 vs 机器学习

```
传统编程                            机器学习
─────────────────────────────────────────────────
输入: 数据 + 规则                   输入: 数据 + 结果
输出: 结果                          输出: 规则（模型）

      ┌─────────┐                      ┌─────────┐
数据 →│  规则   │→ 结果          数据 →│  学习   │→ 模型
      └─────────┘                结果 →│  算法   │
       人工编写                        └─────────┘
```

**示例：垃圾邮件识别**

| 方式 | 做法 |
|------|------|
| 传统编程 | 人工定义规则：包含"中奖"→垃圾 |
| 机器学习 | 给模型大量邮件样本，让它自己学习规律 |

### 机器学习的本质

> **机器学习 = 从数据中学习模式，用于预测或决策**

```python
# 概念性代码
def 机器学习(数据, 标签):
    模型 = 初始化()
    for 轮次 in range(训练轮数):
        预测 = 模型(数据)
        损失 = 计算误差(预测, 标签)
        模型 = 更新参数(模型, 损失)
    return 模型
```

### 关键术语

| 术语 | 含义 | 示例 |
|------|------|------|
| **模型** | 从数据中学到的数学函数 | y = wx + b |
| **参数** | 模型内部学习的值 | w 和 b |
| **超参数** | 人为设定的配置 | 学习率、层数 |
| **特征** | 描述数据的属性 | 邮件长度、关键词数 |
| **标签** | 数据的正确答案 | 垃圾/正常 |

## 2.2 三大学习范式

### 总览

```
┌─────────────────────────────────────────────────────────────┐
│                      机器学习范式                            │
├───────────────────┬───────────────────┬────────────────────┤
│    监督学习        │    无监督学习      │     强化学习       │
│   (有老师)        │    (自学)         │    (试错)         │
├───────────────────┼───────────────────┼────────────────────┤
│ 数据有标签         │ 数据无标签         │ 环境给奖励         │
│ 学习输入→输出映射   │ 发现数据内在结构    │ 学习最优策略       │
├───────────────────┼───────────────────┼────────────────────┤
│ 分类、回归         │ 聚类、降维         │ 游戏AI、机器人      │
└───────────────────┴───────────────────┴────────────────────┘
```

### 监督学习

**有标签的师徒制学习**

```
训练数据                              预测
───────────────────────────────────────────────
邮件1 → "垃圾"    ┐                   新邮件
邮件2 → "正常"    ├─→ 训练 → 模型 ─→  "垃圾"
邮件3 → "垃圾"    ┘
```

| 任务类型 | 输出 | 示例 |
|---------|------|------|
| **分类** | 离散类别 | 垃圾邮件识别、图片分类 |
| **回归** | 连续数值 | 房价预测、销量预测 |

### 无监督学习

**自主发现数据规律**

```
数据（无标签）                        发现的结构
───────────────────────────────────────────────
用户1的行为                           用户群组A
用户2的行为    ─→ 聚类算法 ─→         用户群组B
用户3的行为                           用户群组C
```

| 任务类型 | 目的 | 示例 |
|---------|------|------|
| **聚类** | 发现群组 | 用户分群、图像分割 |
| **降维** | 压缩特征 | 数据可视化、去噪 |

### 强化学习

**在试错中学习最优策略**

```
        ┌──────────────────────────────────┐
        │                                  │
        ↓                                  │
    ┌───────┐    动作    ┌───────┐        │
    │ Agent │ ─────────→ │  环境  │        │
    └───────┘            └───────┘        │
        ↑                    │            │
        │   状态 + 奖励      │            │
        └────────────────────┘            │
                                          │
        目标：最大化累计奖励 ───────────────┘
```

**示例**：
- 游戏 AI：通过试玩学习获胜策略
- 机器人：通过尝试学习行走

## 2.3 模型训练基础

### 数据集划分

```
全部数据 (100%)
├── 训练集 (70-80%)  ← 用于学习
├── 验证集 (10-15%)  ← 用于调参
└── 测试集 (10-15%)  ← 用于最终评估
```

**为什么要划分？**
- 训练集：让模型学习
- 验证集：调整超参数，避免过拟合
- 测试集：模拟真实场景，评估泛化能力

### 损失函数

**衡量预测与真实值的差距**

```
损失 = f(预测值, 真实值)

常见损失函数：
├── MSE (均方误差)     → 回归任务
├── Cross Entropy      → 分类任务
└── Binary Cross Entropy → 二分类
```

### 梯度下降

**找到损失最小的参数值**

```
想象你在山上，要找到最低点：

       损失
         │    ·  ← 起点
         │   ╱╲
         │  ╱  ╲
         │ ╱    ╲·  ← 沿梯度方向下降
         │╱      ╲
         └────────·── → 参数
                 最优解

更新规则：参数 = 参数 - 学习率 × 梯度
```

### 训练循环

```
┌─────────────────────────────────────────────────┐
│                   训练循环                       │
│                                                 │
│   for epoch in range(轮数):                     │
│       for batch in 数据:                        │
│           ① 前向传播：预测 = 模型(输入)          │
│           ② 计算损失：损失 = loss(预测, 标签)    │
│           ③ 反向传播：计算梯度                   │
│           ④ 更新参数：参数 -= 学习率 × 梯度      │
│                                                 │
└─────────────────────────────────────────────────┘
```

## 2.4 模型评估与问题

### 过拟合与欠拟合

```
欠拟合                  适度拟合                过拟合
(学习不足)              (刚刚好)               (死记硬背)

  ·  ·                   ·  ·                   ·  ·
 ·    ·                 · ╱ ·                  ·╱╲·
─────────              ──╱───                 ╱╲╱╲╱
                        ╱

训练差，测试差          训练好，测试好          训练好，测试差
```

| 问题 | 表现 | 原因 | 解决 |
|------|------|------|------|
| **欠拟合** | 训练和测试都差 | 模型太简单 | 增加模型复杂度 |
| **过拟合** | 训练好测试差 | 模型太复杂 | 正则化、增加数据 |

### 常见评估指标

**分类任务**

```
                    预测
                正例    负例
          ┌────────┬────────┐
    正例  │   TP   │   FN   │
实际      │ (真阳性)│ (假阴性)│
          ├────────┼────────┤
    负例  │   FP   │   TN   │
          │ (假阳性)│ (真阴性)│
          └────────┴────────┘

准确率 = (TP + TN) / 全部
精确率 = TP / (TP + FP)  ← 预测为正的准确性
召回率 = TP / (TP + FN)  ← 找出所有正例的能力
F1 = 2 × 精确率 × 召回率 / (精确率 + 召回率)
```

**示例：垃圾邮件识别**

| 指标 | 含义 |
|------|------|
| 精确率高 | 标记为垃圾的确实是垃圾 |
| 召回率高 | 不漏掉任何垃圾邮件 |
